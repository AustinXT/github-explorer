# Phase 3 — 策展内容分析（替换 repo-miner 的代码内容分析）

**目标**：深挖这个资源仓库的策展质量、运营模式、可迁移经验、信息差与风险——产出**判断**而非描述。
**输入变量**：LOCAL_PATH / FULL_NAME / GITHUB_URL / **DETECTED_TYPE**（awesome|learning|atypical）/ FACTS_JSON
+ Phase 1 摘要（作者画像 / 官方文档导流 / 竞品清单 / 关键 Issue）。

**先做**：`Read FACTS_JSON`（重点 `content_scale` / `update_rhythm` / `detected_type`）+ 实际抽样阅读
仓库内容（README、几个顶层目录、若干条目/章节样本）。据 `DETECTED_TYPE` 选下面对应镜头，再走「共用收尾」。

---

## 镜头 A — awesome / 精选合集

1. **策展质量：编辑判断 vs 链接堆砌** —— 抽样 10-20 条目，看是否有**取舍与点评**（每条为什么入选、有无说明/分类/标星），还是无脑堆 star 数。这是 awesome 列表价值的分水岭。
2. **覆盖度与边界** —— 收录范围多广/多专？边界是否清晰（声明收录标准）？相比同主题竞品，是更全还是更精？
3. **更新与失效治理** —— 死链率（Phase 2 抽样）、是否有 CI 自动查链、是否定期 review。失效治理 = awesome 列表的长期质量命脉。
4. **分类逻辑** —— 顶层目录/章节分类是否清晰好查；是否有 TOC/锚点导航。
5. **运营自动化** —— 是否 CI 校验贡献、自动生成 TOC、自动查链（读 `.github/workflows`）。
6. **流量/商业模式** —— README 是否植入 App/课程/赞助/导流（结合 Phase 1 战略意图）。
7. **可迁移性** —— 这套策展+运营+自动化方法能否复刻到别的主题。

## 镜头 B — 学习资料（教程/路线图/课程/书/面试题）

1. **学习路径与前置** —— 是否有清晰的 learning path、阶段划分、前置知识标注？还是知识点散列？
2. **深度与受众** —— 入门科普 vs 进阶硬核？目标受众（零基础/求职者/在职进阶）？
3. **时间投入与「学完能做什么」** —— 是否给出预计耗时；学完能达到什么具体能力（如「刷进大厂」「能独立做 X」）。
4. **配套完整度** —— 是否含代码示例/习题/项目/答案/社区答疑；理论与实践的配比。
5. **内容维护与时效** —— 技术类教程是否跟进版本更新（停滞的教程会教过时知识，重大风险）；多语言翻译是否同步。
6. **商业模式** —— 免费引流到付费课程/书/订阅？纯公益？
7. **可信度** —— 作者权威性（Phase 1）、外部采用证据（被课程/公司/学校引用）。

## 镜头 C — 非典型项目（先细判子类，再分析）

**先据 `content_scale.main_ext` / `top_dirs` / README 细判子类**：配置规范（如 toml）/ 模板·规则库（如 nuclei-templates）/ 文档站源码 / 媒体列表（IPTV/直播源/订阅源）/ 免费服务仓库（免费域名/API/证书）/ 数据集。然后分析：
1. **解决什么问题** —— 这个非代码仓库满足了什么真实需求？
2. **为何用 GitHub 承载** —— 为什么是 GitHub 而非别的平台（免费托管/版本历史/社区分发/抗审查）？这种「非典型用法」本身常是创新点。
3. **客观价值信号** —— 采用量（被多少项目依赖/引用）、规模、更新活跃度、外部认可。
4. **可复用运营模式** —— 用 GitHub 做内容/服务分发的可迁移经验（如「定时自动化提交流水线」「issue 当工单系统」）。
5. **风险（该类特有）** —— 版权/合规（媒体列表、电子书）、服务可持续性（免费服务的成本与跑路风险）、平台依赖（DMCA/封禁）、内容时效。

## 共用收尾（三类都要产出，对齐报告模板）

- **核心价值提炼**
  - 创新之处：1-N 个，每个标 `(新颖度 X/5 × 实用性 X/5)`。资源类创新多在**运营模式/分发方式/自动化策展**，不在技术。
  - 可复用的模式与技巧：读者能直接搬走的运营/策展/自动化模式（命名 + 一句话 + 适用场景）。
  - 关键设计决策：策展层 trade-off（目录结构、更新频率、覆盖边界、版权姿态、社区 vs 沉默）。
- **竞品格局与定位**：用 Phase 1 的同主题竞品做对比矩阵（3-5 维度）；差异化护城河（持续时长+自动化+系列化 / 策展判断力 / 信任）；竞争风险（更全竞品、AI 直接生成、平台下架）；生态定位。
- **套利机会分析**：信息差 / 技术借鉴 / 生态位 / 趋势判断（重点：AI 是否在改变该资源的用户行为，如「AI 直接答题」削弱刷题仓库）。
- **风险与不足**：诚实列——版权灰产、关键人依赖、体积膨胀、链接失效/内容过时、零互动、低质堆砌等。
- **行动建议**：如果你要用它 / 学它（学策展自动化思路，不学版权模式）/ fork 复刻它。
- **质量评级（填项目画像）**：`策展[优秀/良好/一般] 更新[活跃/停滞] 自动化[高/低]`（learning/atypical 按子类微调质量轴，但行键不变，见 report-template.md）。
