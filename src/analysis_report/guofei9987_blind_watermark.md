# 把水印藏进奇异值：950 行盲水印库扛住截图压缩，还能溯源谁泄了密（1.3 万 Star）

> 一句话总结：blind_watermark 是阿里算法工程师郭飞（scikit-opt 作者）用约 950 行 Python 写成的图片盲水印库——把不可见水印用 DWT-DCT-SVD 频域算法藏进图像的奇异值里，提取时无须原图，且能扛住 JPEG 压缩、缩放、截图、旋转、遮挡。它是中文社区的盲水印事实标准、7 年成熟的「小而美 done software」;典型应用是版权保护与「给每个接收者嵌唯一水印」的泄密溯源——后者也带来隐私/合规的双面性。

## 作者视角

### 问题发现

数字水印是个经典学术课题（频域变换 + SVD），但学术实现往往门槛高、不易用。郭飞作为懂频域/SVD/优化算法的工程师，把它做成了人人能 `pip install`、三行 API 上手的极简库。blind_watermark 不是孤立爆款，而是其「隐藏信息」开源主线的一环：配套有 `text_blind_watermark`（文本盲水印，1.8K★，用零宽字符把水印藏进文字）+ `HideInfo`（信息隐藏合集），与本库构成「图片 + 文本」盲水印组合拳;另一条主线是 `scikit-opt`（启发式优化，6.6K★）。

### 解法哲学

**「单一功能做到极致 + 极简易用 + 鲁棒优先」**。算法上选频域（DWT-DCT-SVD）而非 LSB，用奇异值量化换鲁棒性;工程上把复杂的频域变换封装进 `read_img → read_wm → embed` / `extract` 的极简 API + CLI;文档上中英双语 + 攻击演示矩阵，对中文开发者零门槛。代码紧凑（核心 232 行 `bwm_core.py`）但注释比 0.327 偏高——算法型作者的「小代码量却扎实」。

### 背景知识迁移

作者的算法血统（scikit-opt 的优化算法、SciPy 贡献、SVD/频域功底）直接体现在实现里：用 `numpy.linalg.svd` + `cv2.dct` + `pywt.dwt2` 三件套搭频域管线;用 NumPy `stride_tricks.as_strided` 做零拷贝的 4×4 分块;用 1D k-means 做自适应二值化阈值;用 `RandomState(seed)` 的置乱做密码加密。这些都是「懂数值计算的工程师」的手法。

### 战略图景

不追求持续演进，而是把库做成「成熟可依赖的稳定件」——done software。被 HelloGitHub 收录、知乎/CSDN 大量教程围绕它展开，在「盲水印 python」中文搜索里几乎是头部默认答案。同赛道中文小库（linyacool/blind-watermark 傅里叶版 517★ 等）star 量级远低于它——它是**中文盲水印的事实标准实现**。

## 核心价值提炼

### 创新点（工程层面的扎实，而非首创算法）

**1. DWT-DCT-SVD 频域盲水印 + QIM 量化嵌入** — 新颖度 3/5 · 实用性 5/5 · 可迁移性 4/5

核心管线（`bwm_core.py`）：图像 BGR→YUV→补偶数白边 → 每通道 DWT(Haar 小波)取低频 `ca` → `stride_tricks` 切 4×4 块 → 每块 DCT → 密码置乱系数 → SVD → **把 1 bit 水印量化嵌入最大奇异值**：`s[0] = (s[0] // d1 + 1/4 + 1/2*wm) * d1`（d1=36/d2=20 控鲁棒性 vs 失真）→ 重构 → 逆置乱 → IDCT → 反组装 → IDWT。改奇异值而非像素，是抗压缩/缩放/截图的根本。适用：任何需鲁棒不可见水印的场景。

**2. 真·盲提取（无须原图）** — 新颖度 3/5 · 实用性 5/5 · 可迁移性 4/5

提取时同样 DWT→分块→DCT→置乱→SVD，靠**读量化级别**还原 bit：`wm = (s[0] % d1 > d1/2)`。无须原图比对，部署门槛极低——这是 QIM（量化索引调制）的天然优势。

**3. 冗余嵌入 + 多通道平均 + 1D k-means 纠错** — 新颖度 3/5 · 实用性 5/5 · 可迁移性 5/5

水印 bit 被**循环冗余嵌入**到所有块（`wm_bit[i % wm_size]`）× 3 通道;提取时 `extract_avg` 对所有冗余副本求平均（抗局部损坏/噪声），再用 `one_dim_kmeans` 自适应二值化（无固定阈值）。这套「冗余 + 平均 + 自适应阈值」的纠错思路可迁移到任何噪声信道下的信息恢复。

**4. 双密码安全 + 攻击模拟/几何恢复工具链** — 新颖度 2/5 · 实用性 4/5 · 可迁移性 3/5

`password_wm` 置乱水印 bit、`password_img` 置乱 DCT 系数，缺一不可提取;`att.py`（225 行）模拟旋转/裁剪/遮挡/缩放/椒盐噪声/亮度/JPEG 压缩等攻击用于鲁棒性测试，`recover.py`（100 行）做几何攻击后的对齐恢复（裁剪/旋转后填补还原）。

### 可复用模式

1. **频域量化嵌入（QIM on SVD）**：把信息量化进奇异值而非像素 — 鲁棒不可见水印。
2. **冗余循环嵌入 + 多副本平均 + 自适应 k-means 阈值**：噪声信道下的信息纠错恢复 — 任何需抗损坏的编解码。
3. **stride_tricks 零拷贝分块**：NumPy `as_strided` 做高效图像分块 — 图像/矩阵分块处理。
4. **密码置乱做轻量加密**：`RandomState(seed).argsort` 置乱系数/数据 — 简单可逆的数据加密。
5. **极简三段式 API（read → embed/extract）+ CLI**：把复杂算法封装成傻瓜接口 — 任何算法库的易用性设计。

### 关键设计决策

- **选 YUV + 只对低频 ca 嵌入**：转 YUV 后在每通道的 DWT 低频近似系数上嵌入——低频对压缩/缩放更稳定，是鲁棒性的关键选择;补偶数白边处理非整除尺寸，嵌入后裁回原尺寸。Trade-off：低频嵌入鲁棒但容量受限（容量与图片尺寸正相关，#62 报「最多 4.096kb」溢出）。
- **fast_mode 取舍**：`block_add_wm_fast` 跳过置乱、只嵌 s[0]，更快但鲁棒性/安全性略降;默认 slow mode 置乱 + 嵌 s[0]/s[1]。
- **多进程 pool（pool.py AutoPool）**：对每块的 SVD 嵌入/提取并行化，支持批量/大图加速。

## 风险与不足

- **鲁棒性有明确边界（不可神化）**：强在抗几何/压缩/截图，但**怕色相/滤镜类色彩变换**（#136「手机加个滤镜直接全部乱码，目前无解」）、**怕社交软件二次压缩/裁剪**（#175 微信传一遍可能提取失败）。「实验室鲁棒」与「微信传一遍」之间有 gap。
- **容量硬上限**：嵌入容量与图片尺寸正相关，图越小/水印越大越易溢出（#62「最多 4.096kb」）。
- **提取需预知 wm_shape**：提取端必须预先知道水印尺寸/长度，对字符串水印尤其反直觉——是新手第一道坎（#107/#126/#28）。
- **画质 tradeoff**：嵌入强度（d1/d2）越大鲁棒性越强但失真越明显（#57），频域改奇异值并非完全无损。
- **低维护期**：近一年约 1 次提交、作者主导度极高、社区贡献稀薄——是「成熟稳定件」而非「持续演进型」。
- **泄密溯源的 dual-use 与合规（重点）**：给每人嵌唯一水印追溯泄露源，正向是企业防泄密/版权维权;另一面对「群聊截图溯源」等场景意味着**个体在不知情下被追踪**，触及隐私/监控伦理。中文语境下《个人信息保护法》对员工信息处理设限，**「告知—同意」与最小必要原则**是这类落地绕不开的合规线。技术中立，关键在使用方是否透明、是否取得同意、用途是否正当。
