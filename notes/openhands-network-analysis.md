### 产品矩阵 (README 提炼)

1. **Software Agent SDK** - Python 库，定义 Agent 后可本地或云端 1000+ 并发运行
2. **OpenHands CLI** - 最简入口，体验类似 Claude Code / Codex，支持任意 LLM
3. **Local GUI** - REST API + React SPA，体验类似 Devin / Jules
4. **OpenHands Cloud** - 托管版，app.all-hands.dev, 免费支持 Minimax 模型
5. **OpenHands Enterprise** - Kubernetes 私有化部署，源码可见但需商业授权

### 架构演进

- **V0 (旧)**: 单体 FastAPI + Socket.IO, 组件紧耦合
- **V1 (当前)**: 解耦架构，Agent 在隔离沙箱中运行，通过 REST API 通信
- **V0 废弃计划**: 2026-04-01

**OpenHands 差异化优势**:
1. **模型无关** -- 唯一支持 Claude/GPT/DeepSeek/Qwen/Llama/Ollama 等全模型的平台
2. **自托管** -- 企业防火墙内部署，数据不离开内网
3. **Docker 沙箱** -- 默认安全隔离执行
4. **多 Agent 委派** -- 支持并行编排多个 Agent
5. **开源 + 商业双轨** -- MIT 核心 + 企业版授权

### 核心亮点
1. **开源 AI Agent 赛道王者** -- 69.5K Stars 遥遥领先同类开源项目
2. **学术+工业双轮驱动** -- CMU/UIUC 学术团队 + $23.8M 商业融资
3. **模型无关架构** -- 唯一真正不绑定任何 LLM 提供商的平台级产品
4. **V1 架构升级** -- 从单体到解耦沙箱的根本性架构进化，deadline 2026-04-01
5. **Dogfooding** -- openhands-agent 自身已贡献 73 次提交

### 关注风险
1. **架构迁移风险** -- V0->V1 迁移 deadline 仅剩 10 天 (2026-04-01)
2. **单仓库膨胀** -- #10649 讨论拆仓，293MB 磁盘占用，2,637 文件
3. **商业化路径** -- 开源 MIT vs 企业授权的平衡，社区可能分歧
4. **竞争白热化** -- Claude Code, Devin, Cursor, Codex 均在快速迭代
5. **许可证复杂性** -- enterprise/ 目录非 MIT，混合许可可能造成使用困惑

### 适合探索的方向
- AI Agent 架构设计 (事件驱动 + 沙箱隔离)
- 多模型编排与模型无关抽象层
- LLM Risk Analyzer 安全机制
- SWE-Bench 评估方法论与 Agent 轨迹数据
- 开源到商业化 (OSS -> Cloud -> Enterprise) 的产品化路径
