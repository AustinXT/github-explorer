## 生态网络

### 技术依赖生态

**前端**：React 18 + Redux + Material-UI + Vega-Lite
**后端**：Flask + DuckDB（会话级数据引擎）
**AI 集成**：通过 [LiteLLM](https://github.com/BerriAI/litellm) 统一支持 OpenAI、Azure、Anthropic、Ollama 等多种模型
**数据源**：CSV/Excel/JSON 文件、MySQL、PostgreSQL、MSSQL、Azure Data Explorer、Azure Blob、Amazon S3、MongoDB、URL 实时数据

### 下游使用

- Fork 数量高达 1,382，但最高 star 的 fork 仅有 2 star，说明 fork 主要是个人使用/学习目的
- 未发现有重要的二次开发或衍生项目
- 作为微软研究院项目，更多定位为研究原型工具，尚未形成广泛的插件/扩展生态

### 部署方式

- `uvx data_formulator` / `pip install data_formulator`（本地安装）
- Docker 支持（社区贡献 PR #253）
- GitHub Codespaces 支持
- 在线 Demo：https://data-formulator.ai/

### 设计理念

1. **混合交互**：将 GUI 直接操作与自然语言输入融合，用户不需要写完整 prompt，而是通过拖拽 + 简短描述表达意图
2. **概念驱动（Concept-driven）**：用户通过定义"数据概念"来引导 AI，而非纯粹依赖自然语言
3. **透明性**：所有 AI 生成的操作都展示源代码，用户可以验证和修改
4. **数据溯源（Data Provenance）**：完整追踪数据转换的血缘关系
5. **会话隔离**：每个用户独立的 DuckDB 实例，保障数据安全

### Issue 信号总结

1. **LLM 兼容性**是用户最关心的话题（#49, #147, #93）：用户希望自由接入各种第三方 LLM，包括中国的 DeepSeek、LM Studio 本地模型等
2. **数据源扩展**需求强烈（#156, #203）：用户希望连接更多数据库和数据源
3. **图表类型**不够丰富（#123, #193）：v0.7 已大幅扩展到 30 种图表
4. **稳定性/易用性**：API Key 配置、大文件处理、CSV 解析等基础功能仍有改进空间
5. Issue 总量相对较少（48 个），说明项目仍处于早期采用阶段，尚未形成大规模用户基础

## 快速判断

### 一句话总结

微软研究院出品的 AI 数据可视化工具，由顶级 HCI/PL 研究者主导，将多年程序合成研究成果产品化，以"概念驱动+混合交互"的独创方法论解决数据可视化创建问题。

### 优势信号

1. **顶级学术背景**：核心作者 Chenglong Wang 是 CHI/POPL 最佳论文得主，研究方向与项目完美匹配
2. **微软研究院品牌**：Microsoft Research 出品，长期研究投入的保障
3. **快速迭代**：20 个月内从 0.1 迭代到 0.7，功能演进速度惊人
4. **差异化方法论**：概念驱动的混合交互模式有学术论文支撑，非简单的"自然语言转图表"
5. **开放生态**：MIT 协议、多 LLM 支持、在线 Demo 可用
6. **15K+ Star**：社区认可度高

### 风险信号

1. **高度依赖个人**：72% 提交来自 Chenglong Wang，关键人风险极高
2. **研究项目定位**：仍以研究原型为主，社区贡献者数量有限（27 人）
3. **PyPI 下载量低**：月下载不足 1000，实际部署/使用量可能有限
4. **Issue 暴露的稳定性问题**：大文件处理、数据库连接、LLM 兼容性等基础功能仍待打磨
5. **无商业化路径**：作为研究项目，缺乏明确的商业化计划和企业级支持
6. **竞争激烈**：PyGWalker（15.7K Star）同赛道且更成熟，商业 BI 工具也在快速加入 AI 能力

### 适合关注的人群

- 数据分析师：希望用 AI 辅助创建可视化，但不想完全依赖纯自然语言
- HCI/可视化研究者：学习概念驱动交互设计的优秀案例
- AI 应用开发者：参考 LLM 在数据转换+可视化场景的集成方式
- 微软技术生态用户：与 Azure 服务集成，未来可能整合到微软产品线

### 趋势判断

项目处于**快速上升期**，Star 增长稳健（日均 ~4），版本迭代频繁。v0.7 引入的企业级特性（安全加固、Workspace、Data Lake）暗示团队有将其从研究原型推向生产级工具的意图。需要持续观察其能否吸引更多核心贡献者、降低关键人依赖。
