### 技术生态定位

Ladybird 处于一个极其稀缺的生态位——**独立浏览器引擎**。全球目前仅有 3 大浏览器引擎垄断市场：
- Google Blink（Chrome、Edge、Opera、Brave 等）
- Apple WebKit（Safari）
- Mozilla Gecko（Firefox）

Ladybird 是近 20 年来第一个认真尝试从零构建完整浏览器引擎的项目。

### 上游依赖

- FFmpeg（多媒体处理）
- curl（网络传输）
- OpenSSL（加密）
- vcpkg（依赖管理）

### 一句话总结

**Ladybird 是由前苹果 WebKit 工程师 Andreas Kling 与 GitHub 创始人 Chris Wanstrath 联合发起的非营利独立浏览器项目，从零构建全新引擎，WPT 通过率已达 90%+，计划 2026 年发布首个 Alpha 版本，是近 20 年来最有可能打破 Blink/WebKit/Gecko 三足鼎立格局的挑战者。**

### 关注理由

1. **反垄断意义**：浏览器引擎多样性对开放互联网至关重要，Ladybird 是唯一从零构建的独立选项
2. **技术突破**：WPT 90%+ 的成就证明了从零构建可行性
3. **价值观驱动**：拒绝搜索默认协议、拒绝用户数据变现，代表了互联网应有的样子
4. **Rust 迁移先锋**：AI 辅助的 C++ 到 Rust 大规模迁移，对整个行业有示范价值
5. **明星团队**：前苹果工程师 + GitHub 创始人的组合极具说服力

### 风险因素

1. **资金可持续性**：纯捐赠模式在浏览器引擎这种长期重资产项目中面临挑战
2. **网站兼容性**：主流网站（Telegram、YouTube）兼容问题仍需大量工作
3. **扩展生态缺失**：uBlock Origin 等关键扩展尚不支持
4. **移动端缺席**：暂无 Android/iOS 计划，限制用户覆盖面
5. **人才集中风险**：Andreas Kling 个人贡献占 36%，关键人依赖度高
