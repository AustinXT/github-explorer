### 量化结论

1. **中型成熟项目**：4.4 万行 Python 代码、169 个源文件，规模适中，架构清晰（src-layout）
2. **版本驱动节奏**：commit 峰值集中在版本发布月，日常以 fix 和维护为主，体现「大版本集中交付」模式
3. **核心驱动模式**：Piotr Skalski 独占近 30% commit，三人贡献近 60%，Bus Factor 较低
4. **迭代稳定化**：fix 类 commit 占 54%，新功能 feat 仅占 5%，说明 API 已基本定型，当前以完善和修补为主
5. **社区参与度可**：169 个贡献者，但去除 bot 后活跃核心仅 ~15 人，依赖自动化工具有效（pre-commit-ci + dependabot 合计 900+ commits）
6. **发布趋于收敛**：从早期每月一版到近期 4-8 个月一版，版本号趋近 0.28，暗示可能酝酿 1.0 里程碑
