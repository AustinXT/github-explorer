### 关键依赖

核心框架与数据处理：
- **fastnbt / fastanvil** — Minecraft NBT 格式解析与 Anvil 世界文件处理
- **geo** — 地理空间计算（多边形、坐标变换）
- **parquet** — Apache Parquet 列式存储读取（Overture Maps 数据）
- **image** — 图像处理
- **rayon** — 数据并行计算

网络与数据获取：
- **reqwest** — HTTP 客户端（OSM API / Overture Maps API）
- **flate2** — gzip 解压

GUI 框架（可选 feature `gui`）：
- **tauri** — 跨平台桌面 GUI 框架
- **rfd** — 原生文件对话框

Bedrock 版支持（可选 feature `bedrock`）：
- **bedrockrs_level / bedrockrs_shared** — Bedrock 版世界格式
- **nbtx / zip / rusty-leveldb** — Bedrock 相关数据处理

### 关键发现

1. **单人核心驱动**：louis-e 贡献超 77% commits，项目深度依赖主作者精力
2. **代码规模适中**：72K LoC 以 Rust 为主力语言，JavaScript 主要承载 GUI 部分（Tauri 前端）
3. **架构模块化良好**：`element_processing/` 子模块化拆分清晰（建筑、道路、水域、土地利用各自独立）
4. **注释率偏低**：代码/注释比 8:1，Rust 代码 24K 行仅 2.3K 行注释，但内嵌文档（doc comments）有 1,092 行
5. **发布节奏成熟**：从 v2.1 起形成「Feature Update → Quality Update」双轨发布模式，平均 2-3 月一个大版本
6. **爆发式增长**：2025 年 1 月单月 281 commits，是项目转折点，此后保持高强度迭代
7. **跨平台野心**：通过可选 feature flag 同时支持 Java Edition 和 Bedrock Edition
