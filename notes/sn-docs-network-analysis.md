## 5. 竞品识别

### 5.1 直接竞品（开源协作文档）

| 竞品 | 定位 | 技术栈 | Stars | 差异点 |
|------|------|--------|-------|--------|
| **Outline** | 团队知识库 & Wiki | Node.js + React | ~30K | 更成熟、功能更丰富，但不支持离线编辑 |
| **AFFiNE** | Notion 替代品 | Rust + TypeScript | ~45K | 支持白板/数据库，功能更全但更重 |
| **Docmost** | 协作文档 & Wiki | Node.js | ~10K | 更轻量，功能较少 |
| **AppFlowy** | Notion 替代品 | Rust + Flutter | ~60K | 桌面优先，移动端支持好 |

### 5.2 间接竞品（商业/其他开源）

| 竞品 | 类型 | 差异点 |
|------|------|--------|
| **Notion** | 商业 SaaS | 数据库、自动化等全功能，但不可自托管 |
| **Google Docs** | 商业 SaaS | 协作体验最佳，但数据主权问题 |
| **CryptPad** | 开源加密协作 | 端到端加密，但编辑体验较弱 |
| **ONLYOFFICE** | 开源办公套件 | 传统文档格式兼容性好，但非块编辑器 |
| **Etherpad** | 开源实时编辑 | 极简，缺少结构化文档能力 |
| **Collabora Online** | 开源 LibreOffice 云版 | 传统办公格式，非知识管理工具 |

### 5.3 Docs 的差异化优势

1. **政府背书与数据主权**：法德两国政府官方支持，SecNumCloud 安全认证，15 个法国部委已在使用
2. **MIT 许可证**：比竞品更友好的开源许可，明确邀请商业使用
3. **离线编辑**：基于 Yjs CRDT 的离线编辑能力
4. **AI 集成**：内置 AI 重写、摘要、翻译、拼写纠正
5. **导入导出**：支持 .docx、.odt、.pdf 多格式导出

## 9. 综合评估

### 9.1 项目定位

**一句话定位**：法德两国政府联合打造的开源协作文档平台，面向公共机构和企业的 Notion/Google Docs 替代品，强调数据主权和自托管。

### 9.2 优势

1. **强力政府背书**：法国 15 个部委已在使用，德国和荷兰政府正在接入，是真正在大规模生产环境验证的产品
2. **高速增长**：16K+ stars，处于快速上升期，近期媒体曝光度极高
3. **极高开发活跃度**：周均 20+ 提交，版本迭代极快，3 月份已发布 5 个版本
4. **MIT 许可证**：对商业友好，降低采用门槛
5. **技术先进性**：基于 BlockNote.js + Yjs 的现代块编辑器，支持离线、AI 辅助
6. **数字主权叙事**：在欧洲数字主权运动中占据核心位置，政治势能强大

### 9.3 风险/关注点

1. **核心开发者集中**：AntoLC 贡献超 60%，存在关键人风险
2. **品牌命名问题**：Issue #726 讨论重新命名，"docs" 太通用容易与其他项目混淆
3. **功能差距**：相比 Notion 缺少数据库、自动化等高级功能，相比 Outline 功能成熟度略低
4. **自托管门槛**：Docker 构建问题（#757）反映部署体验待优化
5. **GPL 依赖**：部分高级功能（PDF 导出）依赖 BlockNote XL 包（GPL 许可），与 MIT 不兼容，需要特殊构建参数

### 9.4 发展趋势预测

项目处于**快速上升阶段**，受益于欧洲数字主权政策的强力推动。短期内有望突破 20K stars。随着更多欧洲国家采用，项目的长期可持续性有政策保障。技术栈现代、社区活跃、政府资金支持，是 2025-2026 年最值得关注的开源协作文档项目之一。
