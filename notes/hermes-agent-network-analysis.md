## 生态网络

### 上游依赖

- **LLM 提供商**：支持 Nous Portal、OpenRouter（200+ 模型）、z.ai/GLM、Kimi/Moonshot、MiniMax、OpenAI、Anthropic 等——对 LLM 提供商保持中立，不绑定任何单一厂商
- **Honcho**（Plastic Labs）：用户画像建模集成
- **agentskills.io**：技能标准开放协议，技能互操作性
- **mini-swe-agent**：子模块，终端后端

### 下游生态

- **hermes-agent-self-evolution**（245 stars）：自我进化研究分支
- **hermes-paperclip-adapter**（20 stars）：适配器项目
- **atropos**（916 stars）：RL 训练框架，用于训练 Hermes 工具调用模型
- **Hermes-Function-Calling**（1,217 stars）：前身教程项目

### 终端后端支持

Local、Docker、SSH、Daytona、Singularity、Modal——六种终端后端，其中 Daytona 和 Modal 支持 serverless 持久化。

### 消息平台网关

Telegram、Discord、Slack、WhatsApp、Signal、Email——六大消息平台集成，单一 gateway 进程统一管理。

### 竞品/同类项目对比

| 项目 | 定位 | 对比 |
|------|------|------|
| **OpenClaw** | 通用 AI Agent（300K+ stars） | 生态更大（5,000+ skills，50+ 频道），但 Hermes 在模型广度、自改进技能和研究工作流上更强 |
| **Claude Code** | Anthropic 官方 CLI 编码工具 | 更聚焦编码，与 Claude 模型深度绑定；Hermes 在模型自由度和跨平台部署上更灵活 |
| **Cline** | VS Code 内嵌 Agent | 编辑器内体验好，但不支持独立部署和消息平台集成 |
| **Cursor** | AI 编码编辑器 | 商业产品，编码体验更打磨，但缺乏 Agent 自治能力 |
| **Agent Zero** | 自主 Agent 框架 | 类似自主方向，但 Hermes 在 RL 训练和研究集成上更有深度 |
| **LangGraph / AutoGen / CrewAI** | Agent 编排框架 | 面向开发者构建 Agent 的框架层，Hermes 是面向终端用户的完整 Agent 产品 |

**差异化定位**：Hermes Agent 占据了 "自托管、模型无关、自我改进的个人 AI Agent" 这一独特定位——介于 Claude Code 式的编码 CLI 和 OpenClaw 式的消息平台 Agent 之间，且背靠 Nous Research 的模型训练能力形成研究闭环。

### 风险与机会

**风险**：
- 项目年龄极短（公开仅 25 天），star 数可能包含发布期 hype 成分
- 核心贡献者过于集中（teknium1 占 70%），存在 "bus factor" 风险
- 与 OpenClaw 生态差距巨大（300K+ vs 10K），要追赶需持续高投入
- 版本号 0.x，稳定性和向后兼容性尚无保证

**机会**：
- 直接切入 OpenClaw 用户群（内建迁移工具）
- 背靠 Nous Research 的模型训练能力，可形成 "模型 + Agent" 垂直闭环
- 自改进技能和 RL 训练管线是真正的技术护城河
- 模型无关策略降低用户迁移成本
- 媒体报道密集（MarkTechPost、Medium、多个 AI 博客），曝光度高
